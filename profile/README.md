# Action Over Perfection
> 완벽주의를 극복하고, 빠른 실행을 통한 성장을 이루고자 기획한 프로젝트입니다.

### 이 프로젝트를 진행하게 된 계기
무언가를 잘하기 위해 열심히 하려고 하지만, 결국 소극적으로 행동하게 되거나 아예 시작조차 못하게 되는 점이 고민이었어요.<br>
처음에는 개발 실력이 부족하다는 점이 원인이라고 생각했기 때문에 학습에 집중하기도 했지만 제 모습은 이전과 변함이 없었습니다.<br>

그래서 다시 고민해보니, 어쩌면 **완벽하지 못한 상태에서 실행으로 옮기기 두려워하는 태도가 진짜 원인이지 않을까** 라는 생각이 들었습니다.<br>
실행을 해야 결과가 있고, 그 결과를 통해 배움으로써 더 나아진다고 생각했기 때문에 완벽하지 못한 상태라도 일단 실행해보는 습관을 들이기로 했습니다.<br>

### 실행 계획 요약
#### [재정비 후]
👉 [오픈미션 기획을 바꾸게 된 이유](https://github.com/action-over-perfection/logs-and-reflections/blob/main/기획을%20중간에%20바꾸게%20된%20이유.md) 
* 블로그 CRUD API을 구현합니다.
  * 블로그를 웹페이지 형태로 제공합니다.
  * Swagger를 통해 API를 문서화합니다.
* 기능 및 코드를 발전시키면서 **제가 만들 수 있는 가장 최선의 블로그**를 완성하는 것이 목표입니다. 

#### [재정비 전]
* 매일 새로운 프로젝트를 진행합니다.
  * 즉, **하루가 지나기 전에 반드시 프로젝트를 마무리하고 회고를 작성해야 합니다.** (하루 마무리 기준: 취침시간 🛏️)
* 긴 일정이 있는 날에는 이틀에 걸쳐 프로젝트를 진행하거나, 기존 프로젝트를 리팩토링 할 예정입니다.
* 프로젝트 형태는 자바 콘솔 프로그램으로 하되, 상황에 따라 변동될 수 있습니다.
